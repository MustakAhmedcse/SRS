# SalesCom — Current Database Schema (AUTHORITATIVE)

> **This is the team's REAL implemented schema** (PostgreSQL schema `salescomdbtst`), updated 2026-06-18.
> **Where it differs from any other document, THIS file wins.**
>
> **21 tables.** All PKs = `int8 GENERATED BY DEFAULT AS IDENTITY`. All FKs = `int8`.
> Two columns are mixed-case and **must be quoted** in all SQL: `report_setups."IsSetupComplete"` and `final_commissions."Msisdn"`.

## 4. Open items / notes

- `run_stages.sort_order` in the run snapshot corresponds to `section_wise_report_sqls.stage_order` in the setup — different column names, same logical sequence number.
- `report_setups."IsSetupComplete"` and `final_commissions."Msisdn"` — always use double-quotes in SQL; EF Core migrations must emit these with the exact casing.
- POS disbursement has NO RabbitMQ queue — purely Airflow-driven nightly.
- RSO App S2S API endpoints are isolated from the `/api/v1` UI endpoints; they need a separate auth middleware (`application_name` + `secret_key`).
- Monthly commission dump to RSO App: format to be agreed with RSO App team before build.
