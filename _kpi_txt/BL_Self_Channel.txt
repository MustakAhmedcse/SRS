## 1. Folder Overview — BL Self Channel SRF Catalogue
The 'BL Self Channel' folder contains 26 SRFs covering two recurring monthly campaigns run on the BL Self Channel platform via the MyBL App referral programme. Both campaigns run in parallel each month with the same incentive mechanism but targeting different recipient types.
[TABLE]
| Campaign Type | SRF Count | Recipient Type | Months Covered | Requester (early) | Requester (from Feb 2026) |
| Device BP Campaign | 13 | Device Brand Promoters (Device BPs) | Apr 2025 – Apr 2026 | Farzana Akhter, Digital Distribution | Chowdhury Taifur Sadat, Alternate Channel |
| Monoband Campaign | 13 | Monobrand Agents | Apr 2025 – Apr 2026 | Farzana Akhter, Digital Distribution | Chowdhury Taifur Sadat, Alternate Channel |
[/TABLE]
Note: Jul 2025 is absent from Device BP Campaign files (only 12 months visible for Device BP vs 13 for Monoband — the July25 file exists but may have been filed under a different name). All 13 Monoband months are present.
## 2. Common / Universal Conditions (Both Campaign Types)
The following conditions apply identically to both Device BP and Monoband campaigns across all months:
## 2.1 Campaign Mechanism
[TABLE]
| Product Promoted | MyBL App — Banglalink's customer self-service mobile application |
| Campaign Goal | Drive new MyBL App user registrations via referral codes held by BL Self Channel agents |
| Commission Trigger | Successful referral = customer signs up as new MyBL App user AND meets data usage requirement |
| Customer Reward | 1 GB bonus data (valid for 3 days) upon successful registration with referral code |
| Agent Reward | BDT 10 (10tk) cash commission per successful referral |
[/TABLE]
## 2.2 Platform & Data Source
[TABLE]
| App Platform | MyBL App (Banglalink's official customer app) |
| Data Sources | IT SC + IT BI + EV Platform + DMS + POS |
| Registration Data | Provided by IT-DSS (Digital Support System) after campaign end date |
| Referral Data | Provided by IT-DSS after campaign end date |
| Report Maturity Date | 8th of following month (data finalized by this date) |
| Report Published Date | As per Tracker |
[/TABLE]
## 2.3 Campaign Periodicity & Timing
[TABLE]
| Period Type | Monthly — each SRF covers one calendar month (1st to last day) |
| Report Frequency | Ad-hoc; report published at End of Campaign |
| Report Sharing Slot | Both Disbursement and Provision reports issued |
| Product Type | Prepaid AND Postpaid both included |
[/TABLE]
## 2.4 General Technical Conditions
[TABLE]
| Recharge Validity | Valid recharge = EV/i-TOP only |
| Usage Definition | Any usage: MOC, SMSMO, MTC, SMSMT, GPRS, MMS, Partial VAS (per IN CDR) + GPRS recharge + bundle recharge |
| SAF DMS Post Date | Within 7 days of Activation |
| SAF Vetting Date | 14th date of Activation following month |
| Lifting/Issue | N/A — needs activation within 180 days |
| Disconnection | N/A — latest data up to report maturity date |
| Product Exclusions | Any connections EXCEPT Data SIM (MMSTDATA) and EV SIM |
| Other Condition | i-top up (checked under 'Other' in General Conditions) |
[/TABLE]
## 3. Campaign Type 1 — Device BP Campaign
13 SRFs | Apr 2025 – Apr 2026 | Recipient: Device Brand Promoters (Device BPs)
## 3.1 Recipient & Eligibility Conditions
[TABLE]
| Recipient Type | Device Brand Promoter (Device BP) |
| Recipient Description | All Device Brand Promoters |
| List Provisioning | Device BPs list provided by Digital team before campaign start |
| Eligibility Basis | Must be on the approved Device BPs list supplied by Digital team |
| Own Referral Code | Each Device BP is assigned a unique referral code for the campaign period |
[/TABLE]
## 3.2 KPI Definition: Successful MyBL App Referral
[TABLE]
| KPI Name | Successful MyBL App Referral — Device BP |
| Step 1 | Customer signs up on MyBL App using the Device BP's unique referral code |
| Step 2 | Customer must be a NEW user of MyBL App (not previously registered) |
| Step 3 | Customer uses minimum 100 MB data within the first 7 days of the following month |
| Step 4 | Both the sign-up and the referral must occur within the campaign period |
| Commission Trigger | All 4 steps above must be satisfied for a commission to be paid |
[/TABLE]
## 3.3 Commission Structure
[TABLE]
| Commission Rate | BDT 10 (10tk) per successful referral |
| Commission Type | Fixed flat rate — no tiering, no cap stated |
| Currency | Bangladeshi Taka (BDT) |
| Payment Basis | Per successful referral (per qualifying customer sign-up) |
| AIT/Tax Treatment | No AIT deduction mentioned — commission appears to be net BDT 10 |
[/TABLE]
Note: Unlike BL Power SRFs, no AIT (Advance Income Tax) deduction is mentioned. The BDT 10 figure appears to be the direct disbursement amount.
## 3.4 Qualifying Criteria — Step-by-Step
Example from Apr 2025 SRF:
[TABLE]
| Step | Condition | Example |
| 1 | Customer signs up on MyBL App with Device BP's referral code | Customer uses BP's code on 16th Apr 2025 |
| 2 | Customer is a brand new MyBL App user | Never registered on MyBL App before |
| 3 | Customer uses min 100 MB data in first 7 days of next month | Must use 100 MB by 7th May 2025 |
| 4 | Both sign-up & referral within campaign period | Both within 1–30 Apr 2025 |
| Payout | BDT 10 commission to the Device BP | Credited after validation by IT-DSS |
[/TABLE]
Note: The '7 days of next month' window shifted to '8th of next month' from Nov 2025 onward — see Section 5.2 for details.
## 3.5 Referral Code & List Provisioning
[TABLE]
| Unique Referral Code | Each Device BP has their own unique referral code — sign-ups without this code do not qualify |
| List Source | Device BPs list provided by Digital team |
| Validation | Only sign-ups using registered Device BP referral codes are incentivized |
| Data Provision | Registration & successful referral data provided by IT-DSS post-campaign |
[/TABLE]
## 3.6 Data Delivery & Maturity Conditions
[TABLE]
| Maturity Date | 8th of following month |
| Report Published | As per Tracker |
| Data Provider | IT-DSS (after campaign end date) |
| Report Sharing | Disbursement + Provision reports both issued |
[/TABLE]

## 5. Differences & Evolution Across Months
While the core KPI logic is stable across all 26 SRFs, three meaningful changes occurred over the campaign lifecycle:
## 5.1 Ownership Change — Requester & Department (from Feb 2026)
[TABLE]
| Period | Requester Name | Requesting Department |
| Apr 2025 – Jan 2026 | Farzana Akhter | Digital Distribution |
| Feb 2026 – Apr 2026 | Chowdhury Taifur Sadat | Alternate Channel |
[/TABLE]
Note: From February 2026, both campaigns transferred to a new owner (Chowdhury Taifur Sadat) in the Alternate Channel department — reflecting a structural reorganisation of BL Self Channel ownership. The KPI conditions and commission rates remained unchanged.
## 5.2 Data Usage Validation Window Change (from Nov 2025)
The successful referral condition includes a data usage window — when a customer signs up near month-end, they have extra days into the next month to use 100 MB. This window subtly changed:
[TABLE]
| Period | Example Sign-up | Data Usage Deadline | Window Phrasing |
| Apr 2025 – Oct 2025 | 16th of campaign month | 7th of next month | 'by 7th [next month]' |
| Nov 2025 – Apr 2026 | 16th of campaign month | 8th of next month | 'by 8th [next month]' |
[/TABLE]
Note: The shift from '7th' to '8th' as the validation deadline appears from the Nov 2025 SRF onward. This gives customers one additional day to achieve the 100 MB usage — a minor but consistent change.
## 5.3 Reporting Frequency Notation Difference
[TABLE]
| Device BP Campaign — Frequency field | 'Others: End of the campaign' (ad-hoc, post-campaign report) |
| Monoband Campaign — Frequency field | 'Monthly' (standard monthly reporting cycle) |
| Impact | No functional difference to the commission logic — both campaigns run monthly and disburse post-campaign |
| Explanation | Device BP classification as 'ad-hoc' likely reflects that the BP list changes each month and the report is custom-generated; Monoband follows standard monthly reporting |
[/TABLE]
