1. Overview & SRF Coverage
[TABLE]
| Programme | GA Bonanza (+ GA Booster Campaign variant) |
| Total SRFs | 68 SRFs (Aug 2025 – Apr 2026) |
| KPI | GA — Gross Addition (new SIM activations) |
| Recipient Types | 5 types: RSO, RSO Supervisor, DD Manager, Distributor, Region |
| Campaign Windows | Ad-hoc multi-day windows; some monthly (Aug–Nov25); shorter from Dec25 |
| Report Slot | Disbursement — end of campaign window |
| Incentive Evolution | Physical gifts (Aug25–Jan26) → Cash incentives (Feb26+) |
[/TABLE]
GA Bonanza is an ongoing multi-tier incentive programme rewarding field force personnel at every level (RSO, RSO Supervisor, DD Manager) as well as Distributor entities and Regions for achieving GA targets within defined campaign windows. Conditions evolved significantly across the period, with structural changes to gate thresholds, GA attribution rules, incentive types, slab structures, and cluster-specific rules.

4. Gate Conditions Evolution
Both the 48-Hour usage gate and the VLR Active gate are mandatory — both must be met. B2C shares the gate achievement data. Gate thresholds evolved over time.
4.1  48-Hour Usage Gate Threshold
[TABLE]
| Period | Gate Threshold | Applies To |
| Aug–Sep 2025 | >=91% | All clusters / all recipient types |
| Oct–Nov 2025 | >=90% | All clusters / all recipient types |
| Dec25, Jan26 (1-7, 16-25), Feb26 (1-22) | >=80% | All clusters |
| Jan26 (8-15), Jan26 (booster), Feb26 (14+), Mar26, Apr26 | >=85% | All clusters — standard from mid-Jan |
| Apr26 (11-20) | >=85% (all clusters — explicit note added) | Cluster-wide applicability confirmed in SRF text |
[/TABLE]
i  The gate applies as a binary pass/fail — if not met, no incentive regardless of GA achievement. B2C shares the 48hr achievement data separately.
4.2  VLR Active Gate Threshold
[TABLE]
| Period | Gate Threshold | Note |
| Aug–Sep 2025 | >=91% | Same as 48hr gate |
| Oct–Nov 2025 | >=90% | Same as 48hr gate |
| Dec25, Jan16-25, Feb1-22 | >=80% | Same as 48hr gate |
| Jan8-15, Jan booster, Feb14+, Mar, Apr1-10 | >=85% | Standardized threshold |
| Apr26 (11-20) — Standard clusters | >=85% | Binary pass/fail — see Section 4.3 for North Cluster exception |
[/TABLE]
4.3  North Cluster Prorated VLR Rule (Apr26 11-20)
[TABLE]
| Introduced | Apr 2026 (11th–20th Apr) — first appearance in the folder |
| Applies to | North Cluster only; all other clusters use standard binary VLR gate |
| >=85% VLR | 100% of incentive — full payment |
| >=75% to <85% VLR | Prorated incentive as per VLR achievement percentage |
| <75% VLR | Not eligible for any incentive |
| Applies to | RSO, RSO Supervisor, DD Manager, Region SRFs for Apr26 (11-20) window |
[/TABLE]
i  This prorated VLR rule is a structural change from the simple binary gate used in all prior SRFs. It introduces partial incentive eligibility for the North Cluster based on VLR achievement level.

5. RSO GA Definition Evolution
The formula used to count an RSO's GA changed multiple times across the campaign period. These changes affect how BP (Business Partner) GA is treated:
[TABLE]
| Period | RSO GA Formula | BP GA Rule | Cluster Scope |
| Aug–Sep 25 | Market GA + BP GA | BP GA capped at 70% of target; excess above 70% ignored | Central/East/North only; West/NE = Market GA only |
| Oct 25 | Market GA + BP GA | Same 70% cap | Central/Comilla/North; West/CtgN/CtgS/Brahmanbaria/NE = Market GA only |
| Nov 25 | Market GA + RSO Own Code | BP GA removed from formula | All clusters unified |
| Dec25–Jan26 (1-7, 16-25) | Market GA + RSO Own Code | RSO Own Code added | All clusters |
| Jan26 (8-15) | Market GA + RSO Own Code | Same | All clusters |
| Feb26 | Market GA + RSO Own Code | Same | All clusters |
| Mar26 (1-10) original | Market GA + RSO Own Code | Same | All clusters |
| Mar26 (1-10) Revision | Market GA EXCEPT BP GA | BP GA explicitly excluded | All clusters — revised after original filed |
| Mar26 (11-20) | All GA except BP Code GA | BP Code GA excluded | All clusters |
| Apr26 (1-10) | All GA except BP GA & DRC GA | Both BP GA and DRC GA excluded | All clusters — DRC added as new exclusion |
| Apr26 (11-20) | All GA except BP GA & DRC GA | Same | All clusters |
[/TABLE]
i  Mar26 1-10 has both an original SRF and a Revision SRF filed later. The Revision changes the GA formula from 'Market GA + RSO Own Code' to 'Market GA EXCEPT BP GA' — and raises Slab 2 from 1,500tk to 2,000tk.
6. RSO Supervisor GA Definition Evolution
[TABLE]
| Period | RSO Supervisor GA Formula | DRC Treatment |
| Aug–Sep 2025 | RSO GA under supervision + BP GA (with cluster-specific rules same as RSO Sep25) | Not mentioned |

[TABLE]

7.5  Incremental Incentive Rule (Distributor & Region)
[TABLE]
| Rule | 10% incremental incentive for every 5% over achievement above the Slab 2 target |
| Applies to | Distributor SRFs from Feb26 onwards; Region SRFs from Feb26 onwards |
| Example | Slab 2 = BDT 20/GA; if achievement is 110% of Slab 2 target → 10% extra = 22/GA; 115% → 20% extra = 24/GA |
| Does NOT apply to | RSO, RSO Supervisor, DD Manager SRFs — they use fixed slab amounts only |
[/TABLE]
7.6  Combined Phase Retroactive Rule (Apr26 11-20)
The April 2026 (11-20) SRFs introduce a combined Phase 1+Phase 2 incentive option that allows recipients who missed or underperformed in Phase 1 (Apr 1-10) to earn a combined incentive:
[TABLE]
| Phase 1 | Apr 1-10 Apr26 (separate SRF) |
| Phase 2 | Apr 11-20 Apr26 (this SRF) |
| Combined slab | Evaluated on total Phase1+Phase2 GA against combined targets — higher incentive amounts |
| Deduction rule | Combined incentive minus any incentive already received in Phase 1 |
| Eligibility | Only for those who: (a) missed Phase 1 target entirely, OR (b) achieved Slab 1 in Phase 1 but want to try for Slab 2 via combined |
| Exclusion | Who already achieved Slab 1 in Phase 1 cannot claim Slab 1 of combined; who achieved Slab 2 in Phase 1 cannot claim Slab 1 of combined |
| Applies to | RSO, RSO Supervisor, DD Manager, Distributor, Region — all Apr26 (11-20) SRFs |
[/TABLE]

13. Glossary
[TABLE]
| Term | Meaning |
| GA | Gross Addition — new SIM activation |
| FCD | First Call Date — activation gate; SIM must make first call during campaign period |
| VLR | Visitor Location Register — network registry confirming SIM is active on the network |
| 48-Hour usage | Subscriber must have valid usage within 48 hours of activation |
| BP | Business Partner — an external sales agent whose GA can be attributed to an RSO (with limits) |
| RSO Own Code | RSO's personal SIM code — GAs registered under this code counted separately from market GA |
| DRC | Direct Recharge Channel — category of GA excluded from Apr26 formulas |
| MMSTR SIM | A specific SIM type with a lower per-GA incentive rate (appears Apr26 Distributor SRF) |
| DHFF | Direct Hire Field Force — staff directly employed (used as incentive unit for Region SRFs) |
| TCG Bar | Type of account bar placed by carrier; barred connections excluded from GA count from Nov25 |
| Self-barred | Customer-initiated service suspension; excluded from GA count from Feb26 |
| Slab | Achievement tier — different GA targets unlock different incentive levels |
| Max consider | Only 1 incentive per RSO/recipient per campaign (highest applicable slab) |
| AIT | Advance Income Tax — applicable on all cash incentives from Feb26 onwards |
| Combined Phase | Apr26 mechanic: Phase1+Phase2 GA evaluated together for retroactive incentive upgrade |
| Incremental incentive | 10% additional incentive per 5% achievement above Slab 2 target (Distributor/Region) |
| B2C | Business-to-Consumer team at Banglalink — defines targets, shares achievement data for gates |
| T-6 | Report generation date minus 6 days — disconnection cutoff |
| ETSAF | 98% of activations vetted by 7th of following month — settlement condition |
| MNP | Mobile Number Portability — porting from another carrier; counted as valid GA |
| North Cluster | Geographic cluster with prorated VLR rule introduced from Apr26 (11-20) |
[/TABLE]
