1. Campaign Overview & Types
LUS (Low Usage Site) campaigns incentivize field force (RSO), distributors (DD), DD Managers, and retailers to improve performance at BL network sites with low subscriber activity. Campaigns cover multiple KPI types: Gross Add (GA), Deno (denomination) HIT, LSO (Live SIM Out), SSO (Subscriber SIM Out / active subscriber), and FWA (Fixed Wireless Access — WiFi Router).
[TABLE]
| Campaign Type | Recipient | KPI(s) | SRFs | Period |
| Retailer LUS GA Campaign | Retailer | GA | 4 SRFs | Jan–Apr 2026 |
| DD Manager LUS Campaign | DD Manager (Field Force) | GA + Deno + LSO + SSO | 3 SRFs (incl. 1 revision) | Jan–Feb 2026 |
| Distributor LUS GA Campaign | Distributor | GA | 3 SRFs (incl. 1 revision) | Feb, Apr 2026 |
| Distributor LSO & SSO Campaign | Distributor | Active LSO + Active SSO | 2 SRFs | Mar–Apr 2026 |
| Distributor Router/FWA Campaign | Distributor | FWA Router Activation | 3 SRFs | Feb–Apr 2026 |
| RSO Router/FWA Campaign | RSO (Field Force) | FWA Router Activation | 3 SRFs | Feb–Apr 2026 |
| RSO Deno HIT Campaign | RSO (Field Force) | Deno HIT (cumulative) | 1 SRF | Jul 2025 |
| GA Support for DD — New/LUS Sites | Distributor (DD) | GA (slab-based) | 1 SRF | Apr 2025 |
[/TABLE]
2. Section A — Retailer LUS GA Campaign (Jan26–Apr26)
Retailers attached to LUS sites receive monthly GA targets and earn incentives on HIT or Miss basis. Four monthly SRFs exist (Jan–Apr 2026). Conditions are nearly identical across all four, with one key change in the usage gate from Mar26 onward.
A1. Core KPI Conditions (Consistent across Jan26–Apr26)
[TABLE]
| KPI | GA (Gross Add) |
| Target Basis | Retailer-wise target along with BTS mapping shared by B2C |
| Incentive Structure | HIT or Miss — Core market: BDT 40 per GA / Emerging market: BDT 35 per GA |
| Incentive Cap | 200% of target |
| GA Attribution | Biometric BTS preferred; if Biometric BTS code is unknown/null → use FCD BTS; if both null → not counted |
| Dual-Site Tagging | If 2 different sites tagged to 1 GA → consider Biometric site tagging with unique GA |
| Agent List | Last day of each campaign month (e.g., 31st Jan, 28th Feb, 31st Mar, 30th Apr) |
| Target vs Achievement | Retailer-wise target vs achievement along with 48hr data shared by B2C |
| Rounding | General mathematical rounding: below 0.5 rounds down; ≥0.5 rounds up |
| Salescom to EV | Yes (automated) |
| SMS Content | Priyo Bikreta, Apni LUS GA Campaign_[Month] er commission hisabe #### peyeched (AIT katar por). Dhonnobad. |
[/TABLE]
A2. Usage Gate Condition — Key Difference Across Months
The 48-hour usage gate threshold changed from Jan26/Feb26 to Mar26/Apr26:
[TABLE]
| SRF(s) | Usage Gate | Change? |
| Jan26, Feb26 | >=95% usages for 48 hours must be maintained | Baseline |
| Mar26, Apr26 | >=90% usages for 48 hours must be maintained | CHANGED — threshold lowered from 95% to 90% |
[/TABLE]
Note: The 48-hour usage gate is a qualifying condition: retailers must maintain the specified usage rate within 48 hours of GA during the campaign period. Relaxation from 95% to 90% from Mar26 onward.

5. Section D — Distributor LSO & SSO Campaign (Mar26, Apr26)
Distributors earn incentives for achieving Active LSO (Live SIM Out) and Active SSO (active subscriber) targets at their assigned LUS sites. Two SRFs exist (Mar26 and Apr26) with significant structural and definitional changes.
D1. Common Conditions (Both Mar26 and Apr26)
[TABLE]
| KPI | Active LSO & Active SSO (achieved separately per LUS site) |
| Target Basis | LUS-wise targets shared by B2C |
| Eligibility | Minimum 80% sites success rate is mandatory (pro-rated basis) |
| SSO Definition | Active SSO = 2 GA/Pos (LUS-tagged retailer must activate 2 GA to qualify as SSO) |
| LSO Placement Rule | Newly configured & placed EV SIM in selective site is considered for LSO |
| BTS Tagging | Month-end BTS tagging used for site-wise LSO & SSO achievement |
| Incentive Slab | 1–3 sites = BDT 1,000 | 4–9 sites = BDT 2,000 | 10–15 sites = BDT 3,000 | 15+ sites = BDT 4,000 |
| Scope | Distributors should achieve LUS-wise LSO & SSO targets separately (per site) |
[/TABLE]
D2. Key Differences: Mar26 vs Apr26
[TABLE]
| Condition | Mar26 | Apr26 | Changed? |
| LSO Definition | Active LSO = 4 Days Tnx + 200 TK C2S (Core) / 100 TK C2S (Non-Core) | Quality LSO = 500tk C2S + 7 Days C2S | YES — definition and naming changed |
| Target Structure | 2 slabs: minimum slab and maximum slab targets | Single target (no min/max slab structure) | YES — slab structure removed |
| Payout by Slab | Min slab achieved = 80% of allocated amount; Max slab = 100% | Not specified (full amount upon achievement) | YES — 80%/100% split removed |
| Gate (VLR) | Maintain >=80% VLR active | Not explicitly stated in conditions | Removed from details section |
| Campaign Name | Distributor LUS_LSO_SSO_Campaign_LUS_Mar26 | Distributor LUS_LSO_SSO_Campaign_LUS_Apr26 | — |
[/TABLE]
Note: The 'Quality LSO' definition in Apr26 (500tk C2S + 7 Days C2S) represents a higher quality threshold than the Mar26 'Active LSO' (4 Days Tnx + 200/100 TK C2S). This is a meaningful eligibility tightening for LSO.

9. Section H — GA Support for DD: New/LUS Sites (Apr25)
Distributors (DD) earn slab-based GA incentives for delivering gross adds at new BL network sites and LUS sites during April 2025. This is the earliest LUS-related SRF in the folder.
H1. KPI Conditions
[TABLE]
| KPI | GA (Gross Add) — slab-based |
| Recipient | Distributor (DD) |
| Campaign Period | 1–30 April 2025 |
| Target Type | Slab-based GA per LUS site (Single LUS type) |
| Minimum GA | 8 per site |
| Maximum GA | 30 to 80 (dynamic — shared by B2C via target file) |
| Incentive Rate | BDT 200 per GA (incremental, within slab) |
| GA Slab | 8th GA to 30th–80th GA |
| BTS Tagging | Biometric BTS tag used for site-wise GA attribution; if no Biometric BTS tag → FCD BTS considered |
| No Retailer-DD Link | Not required to check retailer code against DD; just BTS-wise GA count vs shared target file |
| Qualified GA Source | BTS-wise qualified GA count shared by B2C in 'G' Column of target file |
| Max Cap Column | F Column in target file used for maximum GA cap |
| Cross-Territory GA | Cross-territory GA conditions apply (as per detail format) |
| Rounding | General mathematical rounding |
| Disconnection | Between FCD and latest data up to report generation date (T-6) |
| Usage | At least 1 outgoing call or any amount of data usage within calendar month |
| Product | Banglalink Prepaid + All MNP (Prepaid) |
| Reactivation | As per reactivation base |
| Maturity Date | 5th of following month (5th May 2025) |
[/TABLE]
Note: Unlike other LUS GA SRFs, this one uses a slab-based incremental model (not per-GA rate with cap) and evaluates at BTS level rather than at retailer level. It also has a distinct usage gate (1 call or any data within month) versus the 48-hour gate in Retailer LUS campaigns.
