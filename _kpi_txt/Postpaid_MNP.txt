Section 1 — Overview: Campaign Types & Channel Summary
The Postpaid MNP folder contains 16 SRFs covering MNP (Mobile Number Portability) and New Customer Acquisition commissions for Postpaid subscribers. All SRFs are from the Alternate Channel and Digital Recharge department and share the same core mechanic: a BDT commission is paid per qualified GA (Gross Acquisition) subject to bundle activation and same-day security deposit.

[/TABLE]

2.2  GA Criteria (New Activation)
[TABLE]
| New Activation: SIM must be activated under subscription type Retail2Postpaid / Banglalink Select Postpaid (M2MADV) & (blkSelPostpaid) with the specific bundle or non-bundle. |
| Security deposit must be equal to or greater than the pack-specific amount, deposited on the same day as activation. |
| Security deposit for New Activation: one-shot deposit (single transaction on activation day). |
| For 200 BDT pack: security deposit amount sourced from DWH (DPOS). |
| Only the specified EQUIPID list (see Section 7) is eligible for GA counting. |
| E-SIM is also counted for achievement with the same conditions as physical SIM (from Oct 2025 onwards). |
| Activation Retailer type must be 'B2C_postpaid' in DMS. |
[/TABLE]
2.3  MNP Criteria
[TABLE]
| MNP: A Prepaid or Postpaid number ported into Banglalink, with final subscription type Retail2Postpaid / Banglalink Select Postpaid (M2MADV) & (blkSelPostpaid), plus the specified bundle or non-bundle. |
| Security deposit for MNP: one-shot or multiple shots, but ALL deposits must be made on the same day as activation. |
| Security deposit must be equal to or greater than the pack-specific amount. |
| For 200 BDT pack: security deposit amount sourced from DWH. |
| From Oct 2025 onwards: deposit can also be done via Banglalink Power App (any registered BL Power App number). DWH source: bl_power_recharge_dwh_yyyymmdd.csv at /ccn1/downstream/export/daily/salescom. |
| E-SIM is also counted for achievement with the same conditions as physical SIM (from Oct 2025 onwards). |
| One-shot recharge will be considered from any source (earlier SRFs); Cumulative recharge or one-shot accepted (Monobrand SRFs). |
[/TABLE]

4.4  Capping Mechanism (Monobrand — applicable from Mar 2025, May 2025)
A per-CCR GA count cap is applied in some months (specifically documented in Monobrand March25 and May25 SRFs):
[TABLE]
| A capping limit (maximum GA count per CCR per month) will be shared with the commission team by end of the commission month. |
| If a CCR achieves more than the capped GA count, the lowest-commission denomination (pack) is deducted from eligibility to align with the cap. |
| CCRs NOT in the target file are exempt from the capping condition — no cap applies to them. |
| The cap is communicated after month-end; it is not a pre-declared fixed number. |
| This capping mechanism was NOT mentioned in Apr25, Aug24, Nov25, or Dec25 SRFs. |
[/TABLE]
