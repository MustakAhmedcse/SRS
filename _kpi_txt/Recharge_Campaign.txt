1.2  Core Recharge Definitions
[TABLE]
| NOTE: These definitions apply uniformly across ALL recharge campaigns unless a specific SRF overrides them. |
[/TABLE]
[TABLE]
| Term | Definition |
| C2C (Charge to Customer) | Transfer/EV recharge from DD/RSO to Retailer. Includes: R2RT, D2RT, SUD12AGENT transaction types. |
| C2S (Charge to Subscriber) | Any amount recharge to subscriber (customer) from retailer. |
| EV (Electronic Voucher) | Electronic voucher recharge — divided into EV Primary and EV Secondary. |
| SC (Scratch Card) | Physical scratch card — divided into SC Primary (lifting) and SC Secondary. |
| Recharge (combined) | EV Secondary + SC Primary + SC Secondary (unless otherwise specified). |
| EV OTF | Electronic Voucher Off-The-Face commission component. |
| IRIS OTF | IRIS system OTF commission component. |
| OTF (total) | EV OTF + IRIS OTF combined. |
| DTR (Daily Trade Return) | Manual + Daily Commission sourced from Salescom system. |
| DRC (Digital Recharge Channel) | Digital recharge method — EXCLUDED from DD achievement unless specifically included. |
| Ryze Recharge | Service Class ID = 141 — EXCLUDED from all Retailer and Distributor campaigns. |
| Postpaid Recharge | SERVICE_CLASS_ID >= 1000 — EXCLUDED from all recharge campaigns. |
| BL Employee Pool Numbers | Banglalink employee numbers — EXCLUDED from all achievement calculations. |
| C2C Achievement (DD/RSO) | Sum of all C2C transactions excluding DRC, BL employee pool, Ryze (141), and Postpaid. |
| C2S Achievement (Retailer) | Sum of all C2S transactions excluding Ryze (141), Postpaid (SC_ID >= 1000), and BL employee pool. |
| AIT (Advance Income Tax) | Applicable deduction for all channel types in Recharge campaigns per NBR regulation. |
| GA (Gross Addition) | New subscriber activation — used as secondary KPI in some DD Manager campaigns. |
| VLR (Visitor Location Register) | Active network usage metric — used as penalty trigger for LSO campaigns. |
| Sprint | Short date window (typically 3–7 days) within a broader campaign period. |
| BSO | Business Support Officer — field officer category under DD Manager. |
| LSO | Local Sales Officer — field officer category under DD Manager. |
[/TABLE]

LSO campaigns mirror the BSO structure (BDT 11,111 / 8,889) but add a VLR-based penalty condition and use a different data source for LSO-category field officers.

[TABLE]

| Secondary KPI | GA achievement vs. target |
| Payout — 100% both | BDT 11,111 |
| Payout — Shortfall | BDT 8,889 |
| VLR Penalty | If VLR (Visitor Location Register) falls below defined threshold, penalty deduction applied |
| VLR Definition | VLR = % of active subscribers on network; low VLR indicates inactive subscriber base |
| Penalty Trigger | VLR < campaign-defined threshold → deduction from earned incentive |
| Penalty Amount | Specified per-SRF (fixed BDT or % of earned incentive) |
| Data Source | RSO Campaign_Recharge_LSO files used for LSO-specific performance data |

[/TABLE]
[TABLE]
| UNIQUE CONDITION — LSO vs BSO: LSO campaigns add a VLR penalty not present in BSO campaigns. VLR measures subscriber activity on Banglalink network. Poor VLR performance reduces the earned incentive. AIT is applied on the net amount after VLR penalty deduction. |
[/TABLE]
4.2  VLR Penalty Rule Detail
[TABLE]
| VLR Condition | Impact | Applicability |
| VLR >= Threshold | No penalty — full earned incentive paid | LSO campaigns only |
| VLR < Threshold | Penalty deducted from earned BDT incentive | LSO campaigns only |

Section 8: RSO C2C Sprint Campaigns
RSO C2C Sprint campaigns use a 3-sprint structure with consecutive date windows. Each sprint is a short window. Achieving each sprint individually earns the base rate; cumulative achievement across sprints can earn higher payout. Special rules apply for missed-sprint catch-up and hybrid/digital outlets.
8.1  Sprint Date Windows — All 6 RSO C2C SRFs
[TABLE]
| SRF File | Sprint 1 | Sprint 2 | Sprint 3 | Notes |
| RSO C2C Campaign_2nd to 7th Aug25 | 2–4 Aug | 5–6 Aug | 7 Aug | 3-day / 2-day / 1-day windows |
| RSO C2C Campaign_9th to 14th Aug25 | 9–11 Aug | 12–13 Aug | 14 Aug | Same structure |
| RSO C2C Campaign_16th to 21st Aug25 | 16–18 Aug | 19–20 Aug | 21 Aug | Same structure |
| RSO C2C Campaign_GAZ_10th_11th and 13th Dec25 | 10 Dec | 11 Dec | 13 Dec | Non-consecutive dates; GAZ variant |
| RSO C2C Campaign_1st to 7th Feb26 | 1–2 Feb | 3–4 Feb | 5–7 Feb | Standard 3-sprint |
| RSO C2C Campaign_Hybrid Distribution_Feb26 | Hybrid dates | — | — | Hybrid/digital outlet variant |
[/TABLE]
8.2  Standard 3-Sprint Payout Logic
[TABLE]
| Sprint KPI | EV C2C volume per RSO per sprint window |
| Sprint 1 Payout | BDT X if Sprint 1 target met (hit-or-miss per sprint) |
| Sprint 2 Payout | BDT Y if Sprint 2 target met (can earn independently of Sprint 1) |
| Sprint 3 Payout | BDT Z if Sprint 3 target met |
| All 3 Sprints Achieved | Higher total BDT (sum of all 3 sprint payouts, possibly with bonus) |
| Consecutive Bonus | Achieving all 3 consecutive sprints may unlock additional BDT |
| Missed Sprint Catch-up | If Sprint 1 missed but cumulative through Sprint 2 meets Sprint 1+2 target, Sprint 1 payout unlocked retroactively |
| GAZ Variant (Dec25) | Non-consecutive dates (10, 11, 13 Dec) — same payout logic applied to non-sequential windows; adjustment rules for date gaps |
| Hybrid/Digital (Feb26) | Hybrid distribution RSOs get BDT 30 per outlet achieving C2C vs. target |
| AIT | Applicable on all sprint payouts |
| Exclusions | DRC excluded from C2C; BL pool excluded; Ryze excluded |
[/TABLE]
[TABLE]
| UNIQUE CONDITION — Sprint Adjustment Rule: If Sprint N is missed but cumulative C2C from Sprint 1 through Sprint N meets the combined target, the individual sprint payout can be awarded retroactively. This prevents penalizing RSOs with uneven daily volumes. |
[/TABLE]
